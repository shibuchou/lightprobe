## 官方评分标准获取状态

截至 2026-06-14，公开访问的赛道页面和赛题列表接口没有返回 `proj40` 的完整任务书、分项分值或评分表。访问赛题详情接口需要赛事官网账号登录；比赛 GitLab 仓库账号不能用于赛事官网登录。

因此，本仓库当前**没有把任何自行整理的功能清单写成官方评分标准**。拿到官方原件后，应按以下方式补齐：

1. 使用本队赛事官网账号登录 [大赛官方网站](https://os.xtnl.org.cn/)。
2. 打开 OS 功能挑战赛道并进入 `proj40` 赛题详情。
3. 下载任务书、评分细则及后续通知原件。
4. 将原件放入 `docs/competition/original/`，文件名保留发布日期或官方版本号。
5. 在本文件增加文件名、来源页面、下载日期和 SHA-256。
6. 依据官方评分项另建“评分项到实现/证据映射表”，但不要改写官方原文。

在官方评分原件补齐前，项目功能完成度请参考：

- `../verification_and_benchmark.md`：项目验证矩阵、压力测试和 benchmark。
- `../member_a_runtime_layout.md`：成员 A 的 runtime、entry/return probe 设计。
- `../../README.md`：项目能力、边界和最小演示命令链。

这些文件是项目实现与验证材料，**不是官方评分标准**。

## 官方来源与复核方法

- 官方网站：https://os.xtnl.org.cn/
- 赛道信息接口：`POST https://os.xtnl.org.cn/index.do`
  - `op=matchlist`
  - `op=getmatch&type=26OS_F`
- 赛题列表接口：`POST https://os.xtnl.org.cn/topics.do`
  - `op=gettopics`
  - `project=2026年全国大学生计算机系统能力大赛-操作系统设计赛(全国)-OS功能挑战赛道`

复核示例：

```bash
curl -fsS -X POST https://os.xtnl.org.cn/index.do \
  -d 'op=getmatch' \
  -d 'type=26OS_F'

curl -fsS -X POST https://os.xtnl.org.cn/topics.do \
  -d 'op=gettopics' \
  --data-urlencode 'project=2026年全国大学生计算机系统能力大赛-操作系统设计赛(全国)-OS功能挑战赛道'
```

接口返回内容可能随赛事阶段更新。准备提交材料前，应再次登录官网核对通知、截止日期和评分原件。
