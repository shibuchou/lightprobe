# 官方开源与诚信要求

以下内容根据官方《全国大学生计算机系统能力大赛操作系统设计赛开源、诚信承诺书》整理。

## 一、参赛身份承诺

- 所有队员均须为在校学生。
- 所有队员均须符合大赛章程规定的报名要求。
- 不得使用不符合参赛资格的人员身份报名或参赛。

## 二、源代码开源要求

- 参赛作品源代码必须遵循开源协议。
- 官方承诺书列举的协议示例包括 GPL-3、Apache、BSD、木兰等开源协议。
- 项目应在仓库中明确写出实际采用的源代码许可证。

## 三、文档开源要求

- 参赛作品文档必须遵循开源协议。
- 官方承诺书指定文档采用 CC BY-SA 4.0 协议。
- 项目技术文档、使用说明和比赛相关公开文档应满足该协议要求。

## 四、仓库公开要求

- 比赛代码仓库在比赛全过程中必须保持公开状态。
- 不得在评审期间临时关闭仓库、改为私有仓库或限制评审访问。

## 五、比赛材料开源范围

以下内容均应开源：

- 比赛代码；
- 技术文档；
- 答辩材料。

## 六、知识产权要求

- 队伍必须遵守知识产权相关法律法规。
- 队伍必须遵守所使用的各种开源协议。
- 不得侵犯他人的知识产权或其他权益。
- 如果发生侵权并造成不良后果，相关法律责任由参赛队自行承担。
- 操作系统设计赛的主办方、承办方和协办方不承担由参赛队侵权行为产生的法律责任。

## 七、学术诚信要求

- 参赛队必须遵守学术诚信。
- 不得存在恶意代码抄袭、技术抄袭或其他学术不端行为。
- 参赛者身份必须符合章程要求。
- 组委会可以组织参赛作品查重检查。

如经组委会确认存在恶意代码或技术抄袭、身份不符合要求等违规行为，参赛队同意服从包括但不限于以下处理：

- 取消参赛资格；
- 取消已经获得的奖项；
- 退回奖金；
- 向参赛队所在学校通报；
- 在大赛网站或公众号公开通报。

## 八、签署要求

1. 承诺书必须打印。
2. 所有参赛队员必须手写签名。
3. 填写赛队名称。
4. 填写签字时间。
5. 将签署后的承诺书扫描。
6. 提交承诺书电子版和手写签字扫描件。

## 九、对 lightprobe 仓库的直接要求

为满足官方承诺，项目至少需要保证：

- GitLab 比赛仓库在比赛全过程保持公开；
- 对外展示使用的 GitHub 仓库同样保持公开；
- 仓库根目录存在清晰的源代码许可证文件；
- 文档明确声明采用 CC BY-SA 4.0；
- 第三方代码、参考实现和依赖保留来源及许可证说明；
- README、技术报告和答辩材料不得包含无法授权公开的内容；
- 所有提交代码能够说明作者和来源，避免复制未授权代码；
- 提交前完成代码与文档的重复内容、许可证和引用检查。

本节是把官方承诺转化为本仓库执行事项，没有改变官方承诺的含义。
