# 比赛要求执行检查表

本检查表根据当前已取得的官方文字要求整理，用于项目提交前自查。它不是新的官方评分标准。

## 队伍资格

- [ ] 所有队员均为符合章程要求的全日制在校学生。
- [ ] 队伍人数为 1 至 3 人。
- [ ] 所有学生来自同一所高校。
- [ ] 每名学生只参加一支队伍。
- [ ] 队伍只报名一个赛道并已选定 `proj40`。
- [ ] 指导教师不超过 2 人。
- [ ] 如进入决赛，至少确定 1 名指导教师。

## 当前时间节点

- [ ] 已确认报名于 2026 年 5 月 28 日前完成。
- [ ] 如需调整成员，只减少人数或更换部分成员，不增加成员、不更换全部成员。
- [ ] 成员调整已经指导教师审核并向比赛邮箱申请。
- [ ] 比赛网站中的最终名单准确。
- [ ] 初赛作品计划在 2026 年 6 月 30 日前提交。

## 报名材料

- [ ] 电子版报名表填写完整。
- [ ] 报名表打印件扫描清晰。
- [ ] 电子版开源、诚信承诺书已准备。
- [ ] 承诺书已由所有队员手写签名。
- [ ] 已准备签字承诺书扫描件。
- [ ] 所有队员学生证 JPG 扫描件齐全。
- [ ] 指导教师工作证 JPG 扫描件齐全。
- [ ] 队长和指导教师手机、邮箱有效。
- [ ] 上传材料能够正常打开和解压。
- [ ] 已收到秘书处邮件或短信确认。

## 开源与许可证

- [ ] 比赛代码仓库保持公开状态。
- [ ] 源代码采用明确的开源许可证。
- [ ] 项目文档声明采用 CC BY-SA 4.0。
- [ ] 比赛代码、技术文档和答辩材料均可公开。
- [ ] 第三方代码和依赖保留来源、版权和许可证信息。
- [ ] 未包含未经授权的代码、图片、文字或其他材料。
- [ ] Git 历史中没有密码、令牌、个人证件或其他敏感信息。

## 学术诚信

- [ ] 所有代码来源可说明、可追溯。
- [ ] 引用参考项目时明确标注来源，没有直接冒充原创。
- [ ] 已进行代码和技术文档重复内容检查。
- [ ] 不存在恶意代码、技术抄袭或身份违规。
- [ ] 所有展示结果和 benchmark 数据均可复现，没有伪造。

## 官方评分资料

- [ ] 已使用赛事官网账号下载 `proj40` 官方任务书。
- [ ] 已下载官方评分细则和补充通知。
- [ ] 官方原件已保存到 `docs/competition/original/`。
- [ ] 已登记原件来源、日期、版本和 SHA-256。
- [ ] 已建立评分项到实现、测试和演示证据的映射表。

最后五项在取得赛事官网登录资料前不能勾选为完成。
